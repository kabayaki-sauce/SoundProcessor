# SECURITY

## 目的
本ドキュメントは、セキュリティ上の懸念や脆弱性の報告手順を定義します。

## 報告窓口

報告は GitHub Issues を使用してください。

- タイトル例: `Security: 脆弱性報告: <概要>`
- 公開Issueであることを前提に、機密情報は直接投稿しないでください。

## 報告時に含める情報

- 影響対象（プロジェクト名、機能、バージョン）
- 再現手順
- 期待動作と実際の動作
- 想定される影響範囲
- 暫定回避策（あれば）
- 実行環境（OS、.NET SDK など）

## 機密情報の扱い

Issue は公開されるため、以下を直接貼り付けないでください。

- シークレット、トークン、認証情報
- 個人情報
- 攻撃再現に直結する過度に具体的な悪用情報

必要最小限の情報で再現性を担保し、追加情報が必要な場合は管理者から連絡します。

## 対応方針

- 受領後、内容確認とトリアージを実施します。
- 影響度（高/中/低）と再現性を基に優先度を決定します。
- 修正方針が定まった時点で、Issue 上で対応状況を更新します。

## 免責

本ポリシーは最善努力での運用を定義するものであり、特定のSLAを保証するものではありません。
